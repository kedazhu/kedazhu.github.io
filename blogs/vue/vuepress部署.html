如何將vuepress搭建的博客部署在github或者gitee上

# 服务器选择

github与gitee都支持一个page功能， Pages 是一个免费的静态网页托管服务，您可以使用 Pages 托管博客、项目官网等静态网页。当然你也可选择自己花钱租服务器来部署自己的项目。gitee与github都可以实现部署页面的功能，这里我选择的是用gitee来部署静态页面，因为giithub是外网，所在访问页面的资源的时候加载会有一点点慢，不过使用gitee需要实名认证比较麻烦（如果要使用gitee的话建议提前取认证，因为审核也是需要花费俩天的时间），所以选择那个部署看个人喜好。

# github部署

  1. 在自己的github (opens new window)上建立博客仓库
点击右上角加号 点击New repository
这里仓库的名字一定按如下格式命名
你取的名字 + .github.io   # 例如kedazhu.github.io
  1. 打包项目
cd .vuepress   # 进入vuepress目录
npm run build  # 打包
cd ./dist # 进入打包好的目录
git init  # 初始化仓库
git add .
git commit -m "提示"
git remote add 连接名字 第一步所建立的博客ssh地址(git@github.com:xxx.io.git)
git push 连接名字 master -u 提交到远程仓库

  1. 打开 https://xxx.github.io， 比如我的https://kedazhu.github.io ，完成！

  2. 注意事项 打包之前需要在.vuepress下的config.js中将baes 改为 "/"不然会出现404现象，找不到依赖的文件

# gitee部署

  1. 在自己的gitee (opens new window)上建立博客仓库
点击右上角加号 点击新建仓库
这里仓库的名字随便起
  1. 打包项目
cd .vuepress   # 进入vuepress目录
npm run build  # 打包
cd ./dist # 进入打包好的目录
git init  # 初始化仓库
git add .
git commit -m "提示"
git remote add 连接名字 第一步所建立的博客ssh地址(git@gitee.com:xxx.io.git)
git push 连接名字 master -u 提交到远程仓库

  1. 进入仓库。点击服务中的gitee page 点击强制使用https，点击更新，稍等一会就会弹出服务器所在地址，点击访问即可

  2. 注意事项 打包之前需要在.vuepress下的config.js中将baes 改为 "/仓库的名字/"不然会出现404现象，找不到依赖的文件

# 常见问题

  1. 前俩节注意事项所说。
  2. Markdown里的图片不显示，因为在markdown里插入的图片使用的是本地的图片，打包的时候这个图片并不是程序依赖的图片，所以就会漏掉它，然后放在服务器上自然就访问不到了，这里推荐一个暴力也比较麻烦的方法: 服务器开启之后就会生成网站的域名，通过域名 + 文件名的方式就可以访问到当前服务器的静态资源如图。
  3. 通过以上方法，通过请求对应url就可以拿到对应的网络资源，所以我们在写Markdown的时候就可以将图片的本地地址改为将要网络地址比如 然后将qwq4.png上传到仓库对应的文件夹中如图 这样解决了图片访问问题。
